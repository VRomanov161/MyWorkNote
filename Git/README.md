### 3 стадии изменений (файлов)

1. Untracked - найден новый файл или изменение

2. После git add, файл переходит в статус Staged

3. После выполнения git commit, файл получает статус copy in local repository, т.е. в БД git есть копия файла (только локально)

4. после git push, стаус меняетя на Copy in Remote repository (например в git gub)

#### вывести историю коммитов

```
# git log

Ключи:

-1 отобразить последний коммит

-p отобразить список изменений
```

#### отменить изменения

```

# git checkout -- [имя_файла]
```

#### отобразить список изменений в файле перед коммитом

```

# git diff --staged
```

#### игнорировать файлы и не следить за их изменениями

```

создаем файл с именем .gitignore

  # mkdir .gitignore

    добавляем файлы или директории которые хотим игнорировать

```
